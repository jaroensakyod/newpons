# Build Brief — แอปติวเคมี สอวน. ค่าย 1 (MVP)

**เดดไลน์: 31 กรกฎาคม (เปิดให้เด็กใช้จริงในเดือน ส.ค. ก่อนสอบ 30 ส.ค.)**
วิธีใช้เอกสารนี้: เปิด Claude Code แล้ววางเนื้อหานี้เป็นบรีฟตั้งต้น ให้มันช่วยสร้างทีละส่วนตามลำดับ

---

## 1. เป้าหมาย MVP

เว็บแอปที่เด็กเข้ามา **ทำข้อสอบเคมีปรนัยเป็นหมวด → รู้ถูก/ผิดทันทีพร้อมเฉลยละเอียด → เห็นว่าตัวเองอ่อนหมวดไหน**

## 2. ต้องมี / ตัดออก (สำคัญ อย่าเกินนี้)

**ต้องมี**
- ล็อกอินง่ายๆ (อีเมล)
- เลือกหมวด → ทำข้อสอบปรนัย
- ตรวจถูก/ผิดทันที + แสดงเฉลยละเอียด (รองรับสมการ/เศษส่วน)
- หน้าสรุปว่าอ่อนหมวดไหน คิดเป็น % ถูกต่อหมวด

**ตัดออกก่อน (เติมทีหลังใน ส.ค.)**
- ตรวจรูปถ่ายลายมือ
- ระบบ Elo (ใช้ % ธรรมดาไปก่อน)
- ระบบ adaptive / แนะนำอัตโนมัติเต็มรูปแบบ
- ตั้งเป้าหมาย + สอบวัดระดับซับซ้อน
- spaced repetition
- ระบบจ่ายเงิน / free-pro
- เสียง / วิดีโอ
- วิชาอื่นนอกจากเคมีค่าย 1

## 3. Tech stack (ล็อกไว้ ไม่ต้องเลือกใหม่)

- **Next.js** (หน้าเว็บ)
- **Supabase** (ล็อกอิน + ฐานข้อมูล Postgres — มี free tier)
- **KaTeX** (แสดงสมการเคมี/เศษส่วน และเฉลยแบบตัดหน่วยด้วย `\cancel` และใส่สีด้วย `\textcolor`)
- **Vercel** (deploy ฟรี ต่อกับ Next.js ได้ทันที)

## 4. โครงสร้างข้อมูล (เวอร์ชันบางสำหรับ MVP)

- **topics** — `id`, `name`, `order` (แค่ 6 หมวด ยังไม่ต้องแตกหัวข้อย่อย)
- **questions** — `id`, `topic_id`, `stem`, `choices` (json 4 ตัวเลือก), `correct_index`, `explanation` (markdown + LaTeX), `difficulty` (1-5)
- **users** — จัดการโดย Supabase Auth
- **attempts** — `id`, `user_id`, `question_id`, `chosen_index`, `is_correct`, `created_at`

> % ถูกต่อหมวด = คำนวณสดจากตาราง `attempts` ไม่ต้องมีตารางแยก

## 5. แผนผังหัวข้อ 6 หมวด (เคมีค่าย 1)

1. ทักษะพื้นฐานปฏิบัติการเคมี (เลขนัยสำคัญ, หน่วย)
2. โครงสร้างอะตอม
3. ตารางธาตุและสมบัติของธาตุ
4. พันธะเคมี
5. **โมลและปริมาณสารสัมพันธ์** ← ออกหนักสุด ทำก่อน
6. แก๊สและสมบัติของแก๊ส

## 6. หน้าจอหลัก

1. **หน้าเข้าสู่ระบบ** — อีเมล (Supabase Auth)
2. **หน้าเลือกหมวด** — การ์ด 6 หมวด + % ถูกล่าสุดของแต่ละหมวด
3. **หน้าทำข้อสอบ** — โจทย์ + 4 ตัวเลือก, กดตอบแล้วเฉลยเด้งทันที (เฉลยเรนเดอร์ LaTeX)
4. **หน้าสรุปจุดอ่อน** — บาร์ % ถูกต่อหมวด เรียงจากอ่อนสุด

## 7. ขั้นตอนผลิตข้อสอบให้เร็ว (คุณตรวจเอง)

1. ให้ AI ร่างข้อสอบทีละหมวด อิงแนวข้อสอบเก่า ออกมาเป็น JSON ตามรูปแบบข้อ 9
2. บังคับให้ AI ตรวจคำตอบตัวเลขด้วยการคำนวณซ้ำก่อนส่ง (ลด error)
3. คุณรีวิว/แก้/อนุมัติ (เพราะแม่นเคมีอยู่แล้ว ทำได้เร็ว)
4. นำเข้า JSON ที่อนุมัติแล้วเข้า Supabase เป็นก้อน
5. ทำหมวดปริมาณสารสัมพันธ์ก่อน แล้วไล่หมวดที่เหลือ

> เป้าคุณภาพ: ~10-20 ข้อ/หมวด (รวม ~60-120 ข้อ) — **น้อยแต่เป๊ะ ดีกว่าเยอะแต่มั่ว** ที่เหลือเติมใน ส.ค.

## 8. วิธีเริ่มกับ Claude Code

1. ติดตั้ง Claude Code (ในแอป Claude เดสก์ท็อป) แล้วเปิดโปรเจกต์ใหม่
2. วางบรีฟนี้ บอกให้ "วางแผนก่อน (plan mode)" แล้วค่อยลงมือ
3. สั่งเป็นภาษาไทยทีละก้อน: (ก) ตั้งโปรเจกต์ Next.js + ต่อ Supabase, (ข) สร้างตารางตามข้อ 4, (ค) หน้าล็อกอิน, (ง) หน้าทำข้อสอบ + เรนเดอร์ LaTeX, (จ) หน้าสรุปจุดอ่อน
4. deploy ขึ้น Vercel
5. ทดสอบกับเด็กจริง 2-3 คน แล้วแก้
