## แหล่งแนวข้อสอบต่างประเทศ (เพิ่ม 17 ก.ค. 2569 — ตามคำขอเจ้าของ)

นอกจากข้อสอบเก่า สอวน. ยังมี `content/archetypes-external/` เก็บ**สรุปสไตล์**ของข้อสอบอื่นที่ใช้
เป็นแรงบันดาลใจแต่งโจทย์ใหม่ได้ (เขียนจากความรู้ทั่วไปเกี่ยวกับสไตล์ข้อสอบ **ไม่ใช่ข้อสอบจริง**
ไม่มีปัญหาลิขสิทธิ์เพราะไม่ได้คัดลอกข้อความจริงจากที่ไหน — จึง commit เข้า git ได้ปกติ):

- `alevel-chemistry.txt` — A-level เคมี (Cambridge/Edexcel/OCR/AQA): โจทย์หลายขั้นต่อเนื่อง
  บริบทอุตสาหกรรม/โลกจริง เน้นเลขนัยสำคัญ
- `bmat-chemistry.txt` — BMAT ส่วนวิทยาศาสตร์: เวลาน้อย ห้ามใช้เครื่องคิดเลข ตอบเร็ว-แม่น
  ระดับพื้นฐานแต่ดักความเข้าใจผิด
- `icho-chemistry.txt` — International Chemistry Olympiad: หลายขั้นมาก ต่อยอดข้อก่อนหน้า
  ใช้ข้อมูลทดลองจริง (ของเราต้องลดทอนความซับซ้อนให้อยู่ในขอบเขตค่าย 1)

**กติกาป้ายสไตล์ (field `style` ใน JSON โจทย์ — ไม่บังคับใส่):**
- ค่าที่ใช้ได้: `"สอวน."` `"A-level"` `"BMAT"` `"IChO"` (ตรวจใน validate-questions.mjs)
- ป้ายนี้บอกว่า **"แต่งเองโดยได้แรงบันดาลใจจากสไตล์ไหน"** ไม่ใช่การอ้างว่าเป็นข้อสอบจริงคำต่อคำ —
  ห้ามใส่ปีจริง (เช่น "สอวน. 68") เพราะนั่นจะเป็นการอ้างว่าคือข้อสอบจริงปีนั้น ซึ่งเราไม่ได้ทำ
- แสดงเป็น badge สีเหลืองใน `docs/review/review.html` และชีท PDF (ให้เจ้าของเห็นว่าข้อไหน
  ตั้งใจให้ยากสไตล์ไหน) — **ไม่ได้โชว์ในแอปที่นักเรียนใช้จริง** (ยังไม่ได้ทำ DB migration ส่วนนี้
  ถ้าอยากให้โชว์ในแอปด้วยภายหลัง ต้องเพิ่มคอลัมน์ `style` ในตาราง `questions` + view `quiz_questions`)
- ข้อสอบเดิม 306 ข้อไม่ได้ย้อนหลังติดป้าย (ทำใหม่ไปข้างหน้าเท่านั้น)

## ขั้นตอน (ไปป์ไลน์ 7 ขั้น)

1. **สกัด text บทที่เกี่ยวข้อง** — `node` + `pdfjs-dist` (ติดตั้งชั่วคราว `npm i -D pdfjs-dist --no-save`)
   ดึงหน้าเป้าหมายเขียนลงไฟล์ชั่วคราวใน %TEMP% (อย่าเก็บในโปรเจกต์)
2. **วิเคราะห์ช่องว่าง (agent อิสระ)** — เทียบ: text หนังสือ + บทเรียนเรา + archetypes + exam-analysis
   → ตารางหัวข้อ: เรามี/มีบางส่วน/ไม่มี × เคยออกสอบ/ไม่เคย → priority ต้องเพิ่ม/ควรเพิ่ม/ข้ามได้
   พร้อม "สาระที่ต้องมี" ต่อหัวข้อ (ข้อเท็จจริง ไม่ใช่สำนวนหนังสือ)
3. **เขียนเนื้อหาใหม่** — จากสาระในข้อ 2 เขียนเป็นภาษาของเราเอง สไตล์เดียวกับบทเรียนเดิม
   (**หลักคิด:**/**ตัวอย่างโจทย์ N**/**วิธีทำ**/**ตอบ**/**ระวัง:**) เลขทุกตัวผ่าน `node -e` ก่อนเขียน
   ห้ามอักษรไทยใน math mode · ถ้าต้องมีรูป → วาด SVG ใหม่เอง (ดูวิธีใน CLAUDE.md)
4. **ตรวจอิสระ (agent ใหม่ สั่งหาข้อผิด)** — คำนวณซ้ำทุกเลข + เช็คข้อเท็จจริง + เช็คว่าไม่ลอกสำนวน
   → แก้จน 0 issue
5. **validate + import** — `npm run validate:lessons && npm run import:lessons`
   (ถ้าเพิ่มโจทย์: validate:questions + import:questions -- --replace)
6. **อัปเดตของที่ generate ต่อ** — ถ้าบทนั้นมี learn-map ให้เช็คว่า heading ใหม่ต้อง map เพิ่มไหม
   → `node scripts/build-worksheet.mjs && node scripts/export-worksheet-pdf.mjs`
   → copy `docs/worksheets/pdf/*.pdf` ไป `public/worksheets/`
7. **ทดสอบเว็บจริง + commit + push** — เปิด /learn/N หรือ /lesson/N ดูจริง console สะอาด
   build ผ่าน → commit พร้อมอัปเดตสถานะใน CLAUDE.md
