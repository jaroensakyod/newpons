# แผนโครงการ: แพลตฟอร์มติวสอบ (ฉบับรวม)

เอกสารนี้รวมทุกอย่างที่คุยกันไว้ให้อ่านที่เดียวจบ และวางเป็นเฟสให้เดินตามได้

---

## 1. วิสัยทัศน์

เว็บแอปคลังข้อสอบ + บทเรียน ที่ทุกคนเข้ามาเรียน ทำข้อสอบ เก็บสถิติ พร้อมเฉลยละเอียด และ**วิเคราะห์ว่าใครอ่อนอะไร ต้องเสริมอะไร** ตามเป้าหมายที่ตั้งไว้ เริ่มจากเคมี สอวน. ค่าย 1 แล้วขยายไปวิชา/สนามอื่น (สอวน. อื่น, เตรียมอุดม, เตรียมทหาร, TGAT/TPAT, รอบ Portfolio และสนามอินเตอร์อย่าง SAT/IELTS/BMAT/IB/AP)

## 2. หลักการสำคัญ 3 ข้อ (ยึดไว้ตลอด)

1. **ออกข้อสอบต้นฉบับเอง อิงแนวข้อสอบจริง — ไม่ลอกของจริงมาลง** ใช้ข้อสอบจริงเป็น "ตัวอ้างอิงแพตเทิร์น" เท่านั้น เลี่ยงความเสี่ยงลิขสิทธิ์ และทำให้ข้อสอบไม่จำกัดจำนวน
2. **ทุกข้อสอบและทุกบทเรียนต้องติดแท็กเข้าหัวข้อในแผนผังทักษะ** นี่คือกระดูกสันหลังที่ทำให้ระบบจุดอ่อน การแนะนำ และ up-level ทำงานได้
3. **คอขวดจริงคือเวลาตรวจของคุณ ไม่ใช่โค้ดหรือค่า API** AI ร่างได้เร็วกว่าที่คนเดียวจะตรวจไหว จัดการด้วยการตั้งเป้าจำนวนตรวจต่อสัปดาห์ เริ่มจากหมวดที่ออกหนักก่อน

## 3. สถาปัตยกรรมระบบ

### 3.1 แผนผังทักษะ + เส้นทาง (learning path)
- แตกแต่ละวิชาเป็นหมวด → หัวข้อย่อย
- เพิ่มความสัมพันธ์ **"ต้องรู้ก่อน" (prerequisite)** ระหว่างหัวข้อ ทำให้แผนผังกลายเป็นแผนที่เส้นทาง
- ใช้ได้ 2 แบบ: (ก) **up-level** — ตั้งเป้าปลายทางแล้วไล่ย้อนว่าต้องปูอะไรก่อน 1-2-3, (ข) **หาต้นตอ** — พลาดปลายทางบ่อย ระบบเช็คย้อนว่ารากปัญหาอยู่หัวข้อต้นน้ำไหน แล้วดันไปเสริมตรงนั้น

### 3.2 ระบบวิเคราะห์จุดอ่อน
- คะแนนความเชี่ยวชาญ (mastery) ต่อหัวข้อย่อย คำนวณด้วย **เรตติ้งแบบ Elo** (ผู้เรียนและข้อสอบมีเรตติ้ง ระบบเรียนรู้ความยากของข้อเองอัตโนมัติ)
- **โปรไฟล์เป้าหมาย**: แต่ละสนามสอบกำหนดน้ำหนักความสำคัญ + เป้าคะแนนของแต่ละหัวข้อ
- **ลำดับที่ต้องเร่ง = (เป้าหมาย − ความเชี่ยวชาญ) × น้ำหนักหัวข้อ**
- **วงจร**: แผนผังทักษะ → สอบวัดระดับ → คะแนนจุดอ่อน → เทียบเป้าหาช่องว่าง → แนะนำบทเรียน/ชุดฝึก → ฝึก → อัปเดต → วนซ้ำ
- เริ่มง่ายก่อน (% ถูกต่อหัวข้อ) → ยกเป็น Elo → ค่อยเพิ่ม spaced repetition และ IRT ทีหลัง

### 3.3 โครงสร้างข้อมูล (รวมทุกฟีเจอร์)
- **subjects** — วิชา
- **topics** — `id`, `subject_id`, `parent_id` (ลำดับชั้น), `name`, `order`
- **topic_prerequisites** — `topic_id`, `requires_topic_id` (เส้นทาง up-level)
- **questions** — `id`, `topic_id`, `stem`, `choices` (json รองรับข้อความ/รูป), `correct_index`, `answer_type` (mcq/numeric/formula/short_text/open), `correct_answer`, `tolerance`, `explanation` (markdown+LaTeX), `difficulty`, `images` (json), `audio_url`, `context_id` (โจทย์ที่อ้างบทความ/บทสนทนาเดียวกัน)
- **contents** — เนื้อหาต่อหัวข้อ: `topic_id`, `lesson_md`, `video_urls` (json), `audio_url`
- **goals** — สนามสอบ/เป้าหมาย + โครงสร้างคะแนนเฉพาะ
- **goal_topics** — `goal_id`, `topic_id`, `weight`, `target`
- **users** — จัดการโดย Supabase Auth
- **attempts** — `user_id`, `question_id`, `answer`, `is_correct`, `self_checked` (กรณีเทียบเฉลยเอง), `time_spent`, `created_at`
- **user_mastery** — `user_id`, `topic_id`, `rating` (Elo), `last_practiced`
- **review_queue** — หัวข้อที่ต้องทบทวน (จากการทำผิด/ตรวจงาน)

## 4. เนื้อหา

### 4.1 ข้อสอบ
- **หาจากไหน**: อ่านของจริง (posn.or.th ฯลฯ) เพื่อจับแนว แล้วออกโจทย์ใหม่เองในสไตล์เดียวกัน
- **รูปในข้อสอบ**: rasterize หน้า PDF เป็น PNG ความละเอียดสูง → vision model ถอดข้อความเป็น LaTeX + ระบุพิกัดรูป → ทางที่ดีที่สุดคือ**วาดใหม่** (ตาราง→HTML, กราฟ→SVG, โครงสร้างเคมี→SMILES/SmilesDrawer) เลี่ยงลิขสิทธิ์และคมชัดกว่า ถ้าวาดใหม่ยากจริงค่อยครอปเก็บใน Supabase Storage
- **ประเภทข้อสอบ 3 ระดับ**:
  - ระดับ 1 — คำตอบตัวเลข/คำสั้น: ตรวจอัตโนมัติ ไม่ใช้ AI (จัดการเลขนัยสำคัญ/หน่วย/สูตรสมมูล/SMILES)
  - ระดับ 2 — แสดงวิธีทำ: เด็กทำเอง พิมพ์คำตอบสุดท้าย (ตรวจด้วยระดับ 1) แล้ว**กางเฉลยละเอียดให้เทียบเอง** ไม่มี AI ตัดสิน
  - ระดับ 3 — AI ตรวจวิธีทำ (พิมพ์/ถ่ายรูป): ทรงพลังแต่พลาดได้ ทำเป็น "ผู้ช่วยแนะนำ" โชว์เฉลยที่ถูกเสมอ ให้เด็กแย้งได้ — เฟสหลัง

### 4.2 บทเรียน (ต้องมีทั้งอ่านและดู)
- **อ่าน**: AI ร่างบทเรียนต้นฉบับต่อหัวข้อ แล้วคุณตรวจ + เสริมแหล่งฟรี (SciMath ของ สสวท., DLTV, OpenStax)
- **ฟัง**: แปลงบทเรียนเป็นเสียงด้วย Thai TTS อัตโนมัติ
- **ดู**: เริ่มจากคัดคลิปฟรีมาฝัง (embed YouTube/SciMath/DLTV) ให้ครบทุกหัวข้อ → ทยอยแทนด้วยคลิปกึ่งอัตโนมัติ (สคริปต์ AI + TTS + สไลด์) → อัดเอง/จ้างเฉพาะหัวข้อสำคัญ

### 4.3 ไปป์ไลน์ผลิต (เร็วเพราะคุณตรวจเคมีเอง)
AI ร่างเป็น JSON ทีละหมวด → บังคับให้ตรวจคำตอบด้วยโค้ดก่อนส่ง → คุณรีวิว/อนุมัติ → นำเข้าเป็นก้อน เริ่มจากปริมาณสารสัมพันธ์ (ออกหนักสุด) หลัก "น้อยแต่เป๊ะ ดีกว่าเยอะแต่มั่ว"

## 5. AI และโมเดล

### 5.1 Model routing (ตอนสร้างเฉลย)
คัดกรองความยากด้วยโมเดลถูก (Flash-Lite/Haiku) → จัดเส้นทาง: ง่าย→โมเดลถูก, กลาง→Flash/Sonnet, ยาก→Pro/Opus → สร้างเฉลย → **ตรวจเลขด้วยโค้ด** → ถ้าพลาด/ไม่มั่นใจ ยกระดับโมเดล + ติดธงให้คนตรวจ (ค่าความยากที่ได้ใช้เป็น difficulty เริ่มต้นด้วย)

### 5.2 ต้นทุน API — สองถัง
- **ถังสร้างเนื้อหา (ครั้งเดียว, ถูก)**: OCR เกือบฟรี (Typhoon) + สร้างเฉลยผ่าน Batch ~$10-30 ต่อ 1,000 ข้อ (5,000 ข้อยังหลักร้อยดอลลาร์)
- **ถังรันตอนใช้งาน (โตตามผู้ใช้)**: ปรนัย/ตรวจตัวเลข = เกือบฟรี (ไม่ใช้ AI); ตรวจรูปถ่าย (vision) = ตัวที่โตจริง ~$0.005-0.015/รูป
- **ตัวช่วยลดค่าใช้จ่าย**: แคชแผนผังหัวข้อ (~90% off), Batch (50% off), เลือกโมเดลตามงาน, สร้างเฉลยล่วงหน้าไม่สร้างสด, งานไทยหนักใช้ Typhoon

### 5.3 ราคาโมเดลอ้างอิง (ต่อ 1 ล้าน token, input/output)
- Claude: Haiku 4.5 $1/$5 · Sonnet ~$3/$15 · Opus 4.8 $5/$25
- Gemini: 2.5 Flash-Lite $0.10/$0.40 · 2.5 Flash $0.30/$2.50 · 3.5 Flash $1.50/$9 · 3.1 Pro $2/$12
- Typhoon (ไทย): API เบต้าใช้ฟรี หรือรันเอง (จ่ายค่า GPU)
- หมายเหตุ: Gemini มี free tier แต่ **นำข้อมูลไปเทรน** — สำหรับข้อมูลเด็ก (อาจเป็นผู้เยาว์) ให้ใช้ tier แบบจ่ายเงิน

## 6. โมเดลธุรกิจ (Free / Pro)
- **Free**: ปรนัยมีลิมิต ต้นทุนต่อคน ~฿3-8/เดือน
- **Pro**: ไม่จำกัด + ตรวจรูป + เฉลยละเอียด + จุดอ่อนเต็มรูปแบบ ต้นทุน ~฿20-40/เดือน
- **ตั้งราคา Pro ~฿149-249/เดือน เก็บผ่านเว็บ** (เลี่ยงค่าหัก App Store 15-30%) คุ้มทุนถ้า conversion ≥ ~5% เผื่อค่าธรรมเนียมชำระเงิน ~3.65%

## 7. เทคโนโลยีและงบสร้าง
- **Stack**: Next.js + Supabase (Auth + Postgres + Storage) + KaTeX (สมการ/เฉลยตัดหน่วยด้วย `\cancel`) + Vercel
- **สร้างด้วย Claude Code** ภายใต้ Claude Max ($100 = 5x / $200 = 20x ต่อเดือน, รวม Claude Code)
- **งบช่วงสร้าง ~2 เดือน: ~$250-500 (~฿8,000-17,000)** ส่วนใหญ่คือค่า Max (ค่ารันตอนมีผู้ใช้จริงเป็นอีกก้อนที่โตทีหลัง)

## 8. แผนการทำเป็นเฟส

**เฟส 0 — เตรียม**: สมัคร Supabase + Vercel, ติดตั้ง Claude Code, ยืนยันแผนผัง 6 หมวดเคมีค่าย 1

**เฟส 1 — MVP เคมีค่าย 1**: ล็อกอิน + ทำข้อสอบปรนัย + เฉลยละเอียด (LaTeX) + หน้าจุดอ่อน (% ถูกต่อหมวด) + ข้อสอบตรวจแล้ว ~60-120 ข้อ

**เฟส 2 — เนื้อหาครบ + เส้นทาง**: เติมข้อครบ 6 หมวด + บทเรียนอ่าน/คลิป(ฝัง)/เสียง ต่อหัวข้อ + เพิ่มเส้น prerequisite (up-level)

**เฟส 3 — จุดอ่อนเต็มระบบ**: ยกเป็น Elo + โปรไฟล์เป้าหมาย + สอบวัดระดับ + spaced repetition + ข้อเขียนระดับ 1 และ 2

**เฟส 4 — ขยายวิชา + เปิดพรีเมียม**: เพิ่ม สอวน. วิชาอื่น / TGAT/TPAT + ระบบ Free/Pro + ตรวจรูปถ่าย (ข้อเขียนระดับ 3)

**เฟส 5 — สนามอินเตอร์**: SAT/CU-AAT/BMAT/AP (ปรนัย เข้าระบบเดิม) → IELTS/CU-TEP (เพิ่ม Listening/เสียง) → IELTS Writing/Speaking (AI ประเมิน) — **ยึดหลักออกข้อสอบต้นฉบับสไตล์เดียวกัน ไม่ลอกของจริง และระวังเครื่องหมายการค้า**

## 9. ความเสี่ยงและการรับมือ
- **ลิขสิทธิ์** → ออกข้อสอบต้นฉบับเอง, รูปวาดใหม่, คลิปฝังไม่ก๊อป, เนื้อหาเขียนเอง
- **เฉลยผิด** → ตรวจด้วยโค้ด + คุณตรวจเอง, "น้อยแต่เป๊ะ"
- **เวลาตรวจของคุณ (คอขวดจริง)** → ตั้งเป้าตรวจต่อสัปดาห์ เริ่มหมวดหนักก่อน ขยายทีละนิด
- **ค่ารันบานปลาย** → ปรนัยเป็นแกน, พรีเมียมเก็บเงิน, ใช้แคช/batch/โมเดลถูก
- **ความเป็นส่วนตัวของเด็ก** → ใช้ tier ที่ไม่นำข้อมูลไปเทรน, แจ้งนโยบายชัด

## 10. ก้าวถัดไปทันที
1. เปิด Claude Code วาง build brief ให้ตั้งโปรเจกต์ Next.js + Supabase (เฟส 1)
2. เริ่มร่างข้อสอบหมวดปริมาณสารสัมพันธ์เป็น JSON แล้วตรวจ — ทำขนานกับการสร้างแอป
3. ให้ AI ร่างเส้น prerequisite เวอร์ชันแรกของ 6 หมวด แล้วปรับ (เตรียมไว้ใช้เฟส 2)
