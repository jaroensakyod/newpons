# มาตรฐานความครบของชีทคณิตศาสตร์ ม.ต้น

เอกสารนี้เป็นเกณฑ์บังคับสำหรับทุกบทถัดไป เพื่อป้องกันปัญหา “มีชื่อหัวข้อครบ แต่เนื้อหาสอนไม่ครบ”

## 1. หลักฐานขอบเขต

ก่อนผลิตทุกบทต้องทำรายการหน่วยการเรียนจากแหล่งหลักสูตร สสวท. และสร้าง coverage matrix โดยแต่ละหน่วยต้องระบุว่าอยู่ในช่วงสอนใด มีตัวอย่างใด มีแบบฝึกข้อใด และมีโจทย์ประยุกต์ใด ห้ามสรุปว่าครบจากชื่อหัวข้อเพียงอย่างเดียว

## 2. เกณฑ์ขั้นต่ำของแต่ละช่วง “สอนแล้วฝึก”

แต่ละช่วงต้องมีครบ:

1. เป้าหมายการเรียนรู้ที่สังเกตหรือวัดได้
2. ความหมายและคำศัพท์สำคัญ
3. หลักการพร้อมเหตุผลหรือแบบจำลอง ไม่ใช่จำกฎอย่างเดียว
4. ตัวอย่างพื้นฐานอย่างน้อย 1 ตัวอย่าง
5. ตัวอย่างที่มีเครื่องหมาย/กรณีพิเศษอย่างน้อย 1 ตัวอย่าง
6. จุดพลาดพร้อมตัวอย่างคำตอบผิด
7. คำถามตรวจความเข้าใจก่อนฝึก
8. แบบฝึกทันทีที่วัดทั้งจำ เข้าใจ และนำไปใช้

## 3. มิติของแบบฝึกทั้งบท

คลังโจทย์ต้องมีอย่างน้อย 6 รูปแบบ ไม่ใช่เปลี่ยนตัวเลขในแม่แบบเดียว:

- คำนวณตรง
- อธิบายหรือเลือกหลักการ
- เปรียบเทียบ/เรียงลำดับ/แทนค่าบนแบบจำลอง
- ตรวจหาข้อผิดพลาด
- เติมจำนวนหรือแก้สมการอย่างง่าย
- โจทย์สถานการณ์หนึ่งขั้นและหลายขั้น

ทุกแนวคิดสำคัญต้องมีโจทย์อย่างน้อย 3 ข้อ และทุกบทต้องมีโจทย์หลายขั้นกับโจทย์ตรวจข้อผิดพลาด

## 4. เกณฑ์เฉลยละเอียด

เฉลยทุกข้อต้องมี:

- หลักคิดที่ตรงกับข้อนั้น
- การแทนค่าหรือแปลงโจทย์
- ขั้นคำนวณจริงจนถึงคำตอบ
- คำตอบพร้อมหน่วยเมื่อมีหน่วย
- วิธีตรวจสอบหรือจุดพลาดเฉพาะข้อ

ห้ามใช้ข้อความแม่แบบ เช่น “ใช้กฎแล้วคำนวณ” โดยไม่แสดงตัวเลขของข้อนั้น

## 5. การตรวจคุณภาพก่อนส่ง

1. ผู้ผลิตทำ coverage matrix และ self-audit
2. ผู้ตรวจอิสระหาช่องว่างเนื้อหา ไม่ใช่ยืนยันรายการเดิม
3. ผู้ตรวจคำนวณทุกข้อใหม่ด้วย Node ก่อนอ่านเฉลย
4. แก้แล้วตรวจซ้ำจน `issues=0`
5. สร้าง PDF และตรวจภาพทุกหน้า
6. เปิดหน้าเว็บจริง ตรวจลิงก์และ console
7. รัน production build

## 6. นิยามคำว่า “ครบ”

บทหนึ่งถือว่าครบเมื่อ coverage matrix ไม่มีช่องว่าง แต่ละช่วงผ่านเกณฑ์ 8 ข้อ แบบฝึกครอบคลุมอย่างน้อย 6 รูปแบบ เฉลยเป็นรายข้อ และการตรวจอิสระไม่มี issue เท่านั้น จำนวนข้อสูงเพียงอย่างเดียวไม่ถือว่าครบ
