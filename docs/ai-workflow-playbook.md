# Playbook วิธีทำงานของ AI ในโปรเจกต์นี้ (สรุปจากที่ Fable 5 ใช้จริง)

เอกสารนี้บันทึก "วิธีทำงาน" ทั้งหมดที่ใช้สร้างแอปนี้ เพื่อให้โมเดลอื่น (เช่น Opus 4.8)
หรือเซสชันใหม่รับช่วงต่อได้ด้วยมาตรฐานเดียวกัน — วางเอกสารนี้เป็น context ตอนเริ่มงาน
หรือชี้ให้อ่านก่อนสั่งงานผลิต/แก้เนื้อหา

---

## 1. หลักคิดหลัก (ใช้กับทุกงานในโปรเจกต์นี้)

1. **อย่าเชื่อผลงาน AI จนกว่าจะยืนยันด้วย 2 ชั้นเสมอ**
   - ชั้นที่ 1: ผู้ตรวจอิสระ (AI อีกตัว/อีกรอบ) ที่ถูกสั่งให้ "หาข้อผิดพลาด" ไม่ใช่ "ยืนยันว่าถูก"
   - ชั้นที่ 2: โค้ด deterministic (สคริปต์ validate, `node -e` คำนวณเลข, KaTeX render จริง)
2. **เลขทุกตัวต้องผ่าน `node -e`** — ทั้งตอนสร้างและตอนตรวจ ห้ามคำนวณในหัวแล้วเขียนเลย
3. **"น้อยแต่เป๊ะ ดีกว่าเยอะแต่มั่ว"** (หลักจากบรีฟ) — คุณภาพมาก่อนจำนวน
4. **ไฟล์ในโปรเจกต์คือ source of truth ไม่ใช่ฐานข้อมูล** — แก้ที่ `content/` เสมอ แล้วค่อย import
   ห้ามแก้ข้อมูลตรงใน Supabase (จะ desync กับไฟล์)
5. **จบงานต้องเห็นของจริง** — `npm run build` ผ่าน + เปิดเบราว์เซอร์ดูหน้าที่แก้ + เช็ค console error
   ก่อนรายงานว่าเสร็จ
6. **งานที่ผู้ใช้ต้องทำเอง** (รัน SQL ใน Dashboard, ปรับ Auth settings) — เตรียม SQL ใส่คลิปบอร์ด
   (`Set-Clipboard`) + ให้ลิงก์ตรงที่ฝัง project id แล้วรอคำยืนยัน อย่าเดาว่าทำแล้ว ให้เช็คด้วย REST จริง

## 2. ไปป์ไลน์ผลิตเนื้อหา (ข้อสอบ / บทเรียน) — หัวใจของโปรเจกต์

ทุกเนื้อหาผลิตด้วยไปป์ไลน์ 4 ขั้น ต่อหมวด (ทำ 6 หมวดขนานกันถ้า spawn agent ได้
หรือทำทีละหมวดเป็นรอบ ๆ ถ้าไม่ได้ — ผลลัพธ์ต้องเท่ากัน):

```
[1] GENERATE  → เขียนไฟล์ลง content/ ตามกติกาข้อ 3
                บังคับ: คำนวณทุกเลขด้วย node -e ก่อนเขียน + ตรวจ JSON parse ก่อนจบ
[2] VERIFY    → ผู้ตรวจอิสระ (ห้ามเห็นบทสนทนาของผู้สร้าง) อ่านไฟล์แล้ว:
                - ข้อคำนวณ: คำนวณใหม่ตั้งแต่ต้นด้วย node -e "โดยยังไม่อ่านเฉลย" แล้วค่อยเทียบ
                - ข้อแนวคิด: ตรวจข้อเท็จจริงเคมี + ไม่มีตัวเลือกถูกซ้อน/กำกวม
                - schema + LaTeX + ความครอบคลุมขอบเขต
                - ถ้าไม่แน่ใจให้ flag ไว้ดีกว่าปล่อยผ่าน → ส่งออกเป็นรายการ issues
[3] FIX       → แก้ตามรายการ issues (เลขที่แก้ต้องยืนยันด้วย node อีกครั้ง)
                ข้อไหนโจทย์เสียเกินแก้ → แต่งข้อใหม่ในหัวข้อย่อยเดิมแทน
[4] RE-VERIFY → ตรวจซ้ำเฉพาะไฟล์ที่ถูกแก้ ด้วยเกณฑ์เดียวกับ [2] จนกว่า issues = 0
```

จากนั้นปิดท้ายด้วยชั้น deterministic เสมอ:

```bash
npm run validate:questions   # หรือ validate:lessons
npm run import:questions     # หรือ import:lessons (ดูข้อ 5)
```

> ผลจริงที่เคยได้: ข้อสอบ 74 ข้อ เจอ issue รอบแรก ~1-4 จุด/หมวด (เช่น เฉลย SO₃ ทั้งที่
> SO₃ ไม่เป็นแก๊สที่ 27°C, อักษรไทยหลุดเข้า math mode, ขาดนิยาม STP) — ทุกจุดถูกจับได้
> ในขั้น VERIFY ก่อนถึงมือเด็ก นี่คือเหตุผลที่ห้ามข้ามขั้นตรวจ

## 3. กติกาเนื้อหา (ใส่ใน prompt ของ GENERATE ทุกครั้ง)

### 3.1 ข้อสอบ (content/questions/NN-slug.json — JSON array)

```json
{
  "topic": "ชื่อหมวดตรงเป๊ะ 1 ใน 6 หมวด",
  "stem": "โจทย์ภาษาไทย (LaTeX ได้)",
  "choices": ["4 ตัวเลือก string"],
  "correct_index": 0,
  "explanation": "เฉลยละเอียดทีละขั้น markdown+LaTeX",
  "difficulty": 3
}
```

- ออกข้อสอบ**ต้นฉบับ**ตามแนว สอวน. — ห้ามลอกข้อสอบจริง
- กระจายความยาก ~20% ระดับ 1-2, 50% ระดับ 3, 30% ระดับ 4-5 และสุ่ม correct_index อย่าให้กอง
- ตัวลวงต้องมาจาก misconception จริง (ลืมคูณจำนวนอะตอม, สลับหน่วย, ลืมหารปริมาตร)
  และใน explanation ให้อธิบายด้วยว่าตัวลวงแต่ละตัวมาจากพลาดแบบไหน
- คำตอบถูกมีตัวเดียว ไม่กำกวม ตัวเลือกห้ามซ้ำกัน

### 3.2 บทเรียน (content/lessons/NN-slug.md — Markdown)

โครงบังคับ: `# ชื่อหมวด` → blockquote ว่าออกสอบแนวไหน → `##` หัวข้อย่อยเรียงพื้นฐาน→ยาก
(แต่ละหัวข้อ: แนวคิด + สูตร + **ตัวอย่างโจทย์พร้อมวิธีทำ** อย่างน้อย 1 ข้อถ้าคำนวณได้)
→ `## 📌 สรุปสูตรและประเด็นต้องจำ` (ตาราง) → `## ⚠️ จุดที่เด็กพลาดบ่อย`
ความยาว ~2,000-3,500 คำ สอนแบบแฟกเตอร์เปลี่ยนหน่วย ใช้ `\cancel` ตัดหน่วยให้เห็น

### 3.3 กติกา LaTeX (เข้มงวด — validator จะ fail ถ้าฝ่าฝืน)

- inline `$...$`, display `$$...$$` เท่านั้น ใช้เฉพาะคำสั่งที่ **KaTeX** รองรับ
  (`\frac \times \cancel \text \mathrm \to \Delta \sqrt \textcolor` ฯลฯ)
- **ห้ามอักษรไทยใน math mode เด็ดขาด** — ข้อความไทยอยู่นอก `$` เสมอ
  ใน `\text{}` ใส่ได้เฉพาะหน่วย/คำอังกฤษสั้น ๆ
- ในไฟล์ **JSON** backslash ต้อง escape เป็น `\\` (เขียน `"\\mathrm{CO_2}"`)
  ในไฟล์ **Markdown** ไม่ต้อง escape
- สูตรเคมีใช้ `$\mathrm{CO_2}$` ไม่ใช่ `CO2` เปล่า ๆ

### 3.4 ค่าคงที่มาตรฐานของโปรเจกต์ (ให้ทุกโจทย์ระบุค่าที่ใช้ไว้ในตัวโจทย์)

อิงตารางค่าคงที่ของข้อสอบจริง: มวลอะตอม H=1, He=4, Li=7, B=11, C=12, N=14, O=16, F=19,
Ne=20, Na=23, Mg=24, Al=27, Si=28, P=31, S=32, Cl=35.5, Ar=40, K=39, Ca=40, Cr=52, Mn=55,
Fe=56, Cu=63.5, Zn=65, Br=80, Ag=108, Sn=118.5, I=127, Ba=137
· STP = 22.4 L/mol (0°C = 273 K, 1 atm) · **R = 0.082** L·atm/(mol·K) · N_A = 6.02×10²³

### 3.4.1 แพตเทิร์นจากข้อสอบจริง (เพิ่ม ก.ค. 2026)

- ข้อสอบจริง 8 ปีอยู่ที่ `ข้อสอบเก่า เคมี ค่าย 1/` — วิเคราะห์รายข้อแล้วใน
  `content/exam-analysis.json` และสรุปแนวข้อต่อบทใน `content/archetypes/NN-*.txt`
- ตอน generate ข้อสอบใหม่: ให้ generator อ่านไฟล์ archetype ของบทนั้น + คลังเดิม
  แล้วออกข้อ "ต้นฉบับ" ตามแนวที่ออกบ่อยและยังขาด — **ห้ามลอก/แปลงเลขจากข้อสอบจริง**
- หมวดจริงมี 8 บท (เพิ่ม สารละลาย ~12%, ของแข็งและของเหลว ~1.5%) น้ำหนักการออก:
  โมล 28% > พันธะ 21% > ตารางธาตุ 14% > สารละลาย 12% > อะตอม 11% > แก๊ส 8% > ปฏิบัติการ 4% > ของแข็งของเหลว 1.5%
- รูปแบบที่ข้อสอบจริงใช้บ่อย: ข้อความ (ก)(ข)(ค) เลือกชุดที่ถูก — ควรมี ~20-30% ของคลัง

### 3.5 คลิปประกอบ (content/lessons/videos.json)

- **ห้ามเดา URL YouTube เด็ดขาด** — ทุกลิงก์ต้องยืนยันก่อนบันทึก:
  `curl -s "https://www.youtube.com/oembed?url=<URL>&format=json"` ต้องได้ JSON มี title
  และ title ต้องตรงหมวดจริง หาไม่เจอ = ใส่ array ว่าง อย่ามั่ว
- เน้นช่องคุณภาพ: สสวท./Proj14, DLTV, TruePlookpanya, ALTV, ช่องติวมีชื่อ (เคมีพี่กัปตัน ฯลฯ)
- แอปฝังแบบ `youtube-nocookie.com` อยู่แล้ว (ความเป็นส่วนตัวของเด็ก) — เก็บ URL ปกติใน json ได้

## 4. โครงระบบ (แตะอะไรได้/ไม่ได้)

- **Stack:** Next.js 16 (Turbopack, `src/proxy.ts` แทน middleware) + Supabase + KaTeX + Tailwind v4
- **กันโกง:** เฉลยไม่ถูกส่งไปเบราว์เซอร์ — client อ่านได้แค่ view `quiz_questions` (ไม่มีเฉลย)
  การตรวจอยู่ใน RPC `submit_answer` (SECURITY DEFINER) ซึ่ง insert ตาราง `attempts` พร้อมกัน
  สถิติมาจาก RPC `topic_stats` — **ห้ามเพิ่ม select policy ตรงบนตาราง questions เด็ดขาด**
- **Supabase project `wvcsrsjnkxikngnlwfwn` ใช้ร่วมกับโปรเจกต์อื่น** (divine-cards/bazi —
  มีตาราง `payment`, bucket `divine-cards`) — แตะได้เฉพาะ `topics/questions/attempts/contents`
- **service_role key ทำ DDL ไม่ได้** — สร้าง/แก้ตารางต้องให้ผู้ใช้รันใน SQL Editor เอง
  (แก้ `supabase/schema.sql` ให้ idempotent เสมอ: `if not exists` / `create or replace` /
  `drop policy if exists` แล้วให้รันทั้งไฟล์ซ้ำได้)

## 5. คู่มือปฏิบัติ (ops รายวัน)

| งาน | คำสั่ง |
|---|---|
| แก้/เพิ่มข้อสอบ | แก้ `content/questions/*.json` → `npm run validate:questions` → `npm run import:questions -- --replace` (คำเตือน: --replace ลบ attempts ของข้อเดิม) |
| แก้บทเรียน/คลิป | แก้ `content/lessons/*.md`, `videos.json` → `npm run validate:lessons` → `npm run import:lessons` (upsert รันซ้ำได้) |
| เช็คตารางมีจริงไหม | REST ตรง: `fetch(url + '/rest/v1/<table>?select=id&limit=1')` ด้วย service key — อย่าเชื่อ supabase-js head request (เคยให้ผลหลอกมาแล้ว) |
| ดู LaTeX เรนเดอร์ | เปิด `/dev/katex-check` |
| ทดสอบเว็บ | dev server ผ่าน `.claude/launch.json` ชื่อ `dev` (**autoPort จำเป็น** — พอร์ต 3000 มักถูกโปรเจกต์ bazi ยึด) บัญชีทดสอบ: `test@chemcamp.local` / `testpass123` |
| build ตรวจ compile | `npm run build` ต้องผ่านก่อนถือว่างานเสร็จ |

## 6. บั๊ก/กับดักที่เจอมาแล้ว (อย่าเสียเวลาซ้ำ)

1. **PostgREST schema cache ช้าหลังรัน DDL** — สร้างตารางแล้ว REST ยัง 404 ได้หลายสิบวินาที
   ให้ retry 2-3 รอบก่อนสรุปว่าตารางไม่มี (และ error `PGRST205` พร้อม hint ชื่อตารางอื่น
   = ตารางไม่มีจริง ไม่ใช่ cache)
2. **พอร์ต 3000 ชน** — โปรเจกต์อื่นของผู้ใช้ (bazi-sft-dataset) ใช้อยู่ ห้าม kill process นั้น
   ใช้ `autoPort: true` ใน launch.json แล้วรับพอร์ตที่ระบบแจก
3. **Next 16:** `middleware.ts` deprecated → ใช้ `src/proxy.ts` (ฟังก์ชันชื่อ `proxy`);
   `params` เป็น Promise ต้อง `await`; pin TypeScript `^5.9` (v7 ยังเสี่ยง)
4. **spawn agents จำนวนมากอาจชน session limit** — ออกแบบงานให้ resume ได้เสมอ
   (ไฟล์เขียนลง disk ทีละหมวด + รู้ว่าหมวดไหนผ่านขั้นไหนแล้ว) และเช็คผลจริงบน disk
   ก่อนสรุปว่าอะไรเสร็จ/ไม่เสร็จ
5. **Windows/PowerShell:** heredoc ใช้ `@"..."@` (ปิดที่คอลัมน์ 0), ไม่มี `&&` ใน PS 5.1,
   node บน Git Bash มี assertion error ตอนจบ process (`grep -v "Assertion failed"` ทิ้งได้
   ไม่กระทบผลลัพธ์)
6. **อีเมลยืนยัน Supabase:** MVP ปิด "Confirm email" ไว้ (Auth → Sign In/Up → Email)
   ถ้าเปิดกลับ เด็กสมัครแล้วต้องกดลิงก์ในเมลก่อนถึงเข้าได้
