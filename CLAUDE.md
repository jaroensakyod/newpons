## กติกาที่ห้ามฝ่าฝืน

1. **เนื้อหาทุกชิ้นต้องผ่านไปป์ไลน์ 4 ขั้น**: สร้าง → ตรวจโดยผู้ตรวจอิสระ (สั่งให้หาข้อผิด ไม่ใช่ยืนยัน)
   → แก้ → ตรวจซ้ำจน issues = 0 แล้วปิดด้วย `npm run validate:*` เสมอ
2. **เลขทุกตัวคำนวณด้วย `node -e` ก่อนเขียน** — ทั้งตอนสร้างและตอนตรวจ (ตรวจ = คำนวณใหม่ตั้งแต่ต้นโดยไม่ดูเฉลยก่อน)
3. **LaTeX**: เฉพาะคำสั่งที่ KaTeX รองรับ, **ห้ามอักษรไทยใน math mode**, ใน JSON escape เป็น `\\`
4. **ห้ามลอกข้อสอบจริง** — ออกต้นฉบับตามแนว สอวน. เท่านั้น
5. **ลิงก์ YouTube ห้ามเดา** — ยืนยันผ่าน oembed (`curl youtube.com/oembed?url=...`) ก่อนบันทึกทุกลิงก์
6. **แก้เนื้อหาที่ไฟล์ `content/` เท่านั้น** แล้ว import — ห้ามแก้ข้อมูลตรงใน Supabase
7. **ห้ามเพิ่ม select policy บนตาราง `questions`** — เฉลยต้องไม่ถึงเบราว์เซอร์
   (client ใช้ view `quiz_questions` + RPC `submit_answer`/`topic_stats` เท่านั้น)
8. **Supabase project นี้ใช้ร่วมกับโปรเจกต์อื่น** (มีตาราง `payment`, bucket `divine-cards`)
   — แตะได้เฉพาะ `topics/questions/attempts/contents` · service key ทำ DDL ไม่ได้
   ตารางใหม่ต้องให้ผู้ใช้รัน SQL เอง (เขียนให้ idempotent, copy ใส่คลิปบอร์ดให้)
9. **จบงานต้อง**: `npm run build` ผ่าน + เปิดหน้าเว็บที่แก้ดูจริง + console ไม่มี error
10. ข้อสอบ/บทเรียนเป็น **draft จนกว่าเจ้าของ (แม่นเคมี) รีวิว** — ย้ำเรื่องนี้เมื่อส่งมอบเนื้อหาใหม่

## คำสั่งประจำ

```bash
npm run dev                # dev server (พอร์ตอัตโนมัติ — 3000 ถูกโปรเจกต์อื่นใช้ ห้าม kill)
npm run build              # ต้องผ่านก่อนถือว่าเสร็จ
npm run validate:questions && npm run import:questions -- --replace   # ข้อสอบ (--replace ลบ attempts เดิม)
npm run validate:lessons   && npm run import:lessons                  # บทเรียน (upsert รันซ้ำได้)
```
