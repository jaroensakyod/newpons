# สรุปโปรเจกต์: ระบบสร้างข้อสอบ/แบบฝึกหัดคณิตศาสตร์อัตโนมัติ (อิงหลักสูตร สสวท)

## เป้าหมายโปรเจกต์
สร้างโปรแกรมที่ผลิตข้อสอบ/แบบฝึกหัดคณิตศาสตร์พร้อมรูปภาพประกอบ (กราฟ, แผนภูมิ, แผนภาพเรขาคณิต) โดยไม่ต้องวาดเอง สามารถสุ่มค่าได้เพื่อสร้างหลายเวอร์ชัน จัดเรียงความยากง่าย และวิเคราะห์ผลนักเรียนเพื่อดูจุดที่ต้องเสริม

---

## 1. เครื่องมือที่ใช้: Python + matplotlib

**เหตุผลที่เลือก:** ผลิตจำนวนมากแบบสุ่มค่าได้ง่ายที่สุด, ฟรี, เชื่อมกับระบบแท็ก/วิเคราะห์ผลได้ในโค้ดเดียวกัน, ไม่ต้องเรียนรู้ syntax ยากแบบ LaTeX/TikZ

**เทียบกับตัวเลือกอื่น:**

| เครื่องมือ | จุดเด่น | ทำไมยังไม่เหมาะเป็นตัวหลัก |
|---|---|---|
| TikZ/LaTeX | สวยระดับสำนักพิมพ์ | เรียนรู้ยาก debug ยาก |
| GeoGebra (ลากวาง) | โต้ตอบสด ไม่ต้องเขียนโค้ด | ผลิตจำนวนมากแบบสุ่มค่าอัตโนมัติไม่ถนัด |
| GeoGebra (GGBScript ผ่าน node-geogebra) | คำสั่งสำเร็จรูปสำหรับเรขาคณิตแบบวงเวียน-ไม้บรรทัด (Circle, Perpendicular, Intersect) | ต้องใช้ headless Chromium หนักกว่ามาก, รันในแซนด์บ็อกซ์นี้ไม่ได้ (ติด network restriction ของ geogebra.org), เหมาะเป็นตัวเสริมเฉพาะโจทย์เรขาคณิตที่ matplotlib คำนวณพิกัดเองยาก ไม่ใช่ตัวแทน matplotlib |
| Manim | วิดีโอเคลื่อนไหวสวย | ออกแบบมาสำหรับวิดีโอ ไม่ใช่ภาพนิ่ง render ช้า |
| Desmos API | ใช้ง่ายบนเว็บ | ไม่เหมาะกับ pipeline ผลิตไฟล์ภาพจำนวนมากแบบออฟไลน์ |

**ข้อจำกัดของ matplotlib ที่ต้องรู้:**
- ภาพประกอบสมจริง (ไอคอนผลไม้, นาฬิกา, เหรียญ) ฟอนต์/emoji มักเพี้ยน ควรใช้ไฟล์ไอคอนแทน
- แผนภาพเรขาคณิตซับซ้อนต้องคำนวณพิกัดเอง ไม่มีคำสั่งสำเร็จรูปแบบ GeoGebra
- วาดได้แค่ภาพนิ่ง ไม่ใช่ interactive

## 2. เรื่องลิขสิทธิ์ (ไม่ใช่คำแนะนำทางกฎหมาย)

- ห้ามคัดลอกโจทย์/รูปภาพจากหนังสือที่มีลิขสิทธิ์ (รวม สสวท เอง) มาแจกจ่ายหรือใช้คำต่อคำ
- ใช้โจทย์เดิมเป็น "แรงบันดาลใจ" ได้ (ดูรูปแบบ/ระดับความยาก/หัวข้อ) แล้วเขียนโจทย์ใหม่เองทั้งหมด
- กฎหมายไทยมาตรา 32 มีข้อยกเว้นเพื่อการศึกษาแบบไม่แสวงกำไร แต่ขอบเขตจำกัดเฉพาะเท่าที่จำเป็น หากจะแจกจ่ายวงกว้าง/เชิงพาณิชย์ควรตรวจสอบเพิ่มเติมหรือติดต่อ สสวท โดยตรง

## 3. ระบบแท็กคลังข้อสอบ

ไฟล์ `question_bank.json` — แต่ละข้อมี field:
- `topic`, `subtopic` — หัวข้อ/หัวข้อย่อย
- `difficulty` — ระดับ 1 (ง่ายสุด) ถึง 4 (ยากสุด)
- `objective` — ตัวชี้วัด (เป็นตัวอย่าง ต้องตรวจกับเอกสารจริงของ สสวท ก่อนใช้งานจริง)
- `question_text`, `answer` — โจทย์และเฉลย
- `generator`, `params` — ชื่อฟังก์ชันวาดรูป + พารามิเตอร์ (เชื่อมกับสคริปต์วาดภาพ)

ไฟล์ `bank_manager.py` ทำหน้าที่:
- กรองข้อสอบตามหัวข้อ/ระดับความยาก
- ประกอบชุดข้อสอบง่าย→ยากอัตโนมัติ (`build_progressive_test`)
- เรียกฟังก์ชันวาดรูปตามที่แท็กไว้ ให้อัตโนมัติ
- พิมพ์แบบฝึกหัด + เฉลยแยกท้าย

**หลักการสำคัญ:** แยก "ข้อมูล" (json) ออกจาก "โค้ดตรรกะ" (py) — เพิ่มข้อสอบใหม่แค่เติมใน json

## 4. แนวทางจัดระดับความยากให้เทียบเท่า สสวท

ตัวอย่างการไล่ระดับ (พาราโบลา):
1. ระดับ 1: a=1, b=c=0 (รูปแบบง่ายสุด)
2. ระดับ 2: a,b,c เป็นจำนวนเต็มทั่วไป
3. ระดับ 3: มีเศษส่วน/ทศนิยม
4. ระดับ 4: โจทย์ปัญหาบริบทจริง

ควรอ้างอิง "ตัวชี้วัดและสาระการเรียนรู้แกนกลาง" ของ สสวท แต่ละชั้นปีเป็นกรอบ (เอกสารตัวชี้วัดใช้อ้างอิงวางแผนได้ปกติ ต่างจากการคัดลอกโจทย์)

## 5. ระบบวิเคราะห์ผลนักเรียน (แนวทาง ยังไม่ได้ลงมือสร้างจริง)

Pipeline ที่ออกแบบไว้ (ดูแผนภาพที่แสดงในแชท):
1. **คลังข้อสอบ** (แท็กหัวข้อ/ระดับ/ตัวชี้วัด)
2. **สร้างชุดข้อสอบ** เรียงง่าย→ยาก
3. **นักเรียนทำข้อสอบ** → บันทึกถูก/ผิดรายข้อ
4. **วิเคราะห์ผล** → หาจุดอ่อนตามหัวข้อ
5. **เสนอแบบฝึกหัดเสริม** เจาะจงหัวข้อที่ยังอ่อน → วนกลับข้อ 2

หัวใจคือบันทึก: รหัสข้อ, คำตอบนักเรียน, ถูก/ผิด, เวลาที่ใช้ แล้ว group by หัวข้อ เพื่อดูภาพรวมจุดอ่อน ข้อมูลจำนวนไม่มากใช้ CSV/SQLite พอ ถ้าเป็นเว็บแอปหลายคนใช้พร้อมกันค่อยขยับไปฐานข้อมูลจริง

**ยังไม่ได้ลงมือสร้าง** — เป็นขั้นต่อไปที่แนะนำให้ทำ

## 7. ข้อบกพร่องที่พบและแก้ไขแล้ว (รอบตรวจสอบล่าสุด)

| ปัญหา | สาเหตุ | วิธีแก้ |
|---|---|---|
| เส้นขนาน: จุดตัด/เลขมุมผิดตำแหน่งจริง | เขียนพิกัดจุดตัดแบบ hardcode ไม่ได้คำนวณจากสมการเส้นตรง | คำนวณจุดตัดจริงจากสมการ `x(y) = x₀ + (y−y₀)/slope` แล้ววางเลขมุมอิงจุดตัดจริง |
| โจทย์ลูกบอล (พาราโบลาบริบทจริง) กราฟผอมผิดสัดส่วน, ป้ายทับหัวเรื่อง | บังคับ `aspect="equal"` ทั้งที่สองแกนคนละหน่วย (วินาที vs เมตร) | เลิกบังคับ aspect เมื่อแกนคนละสเกล, ปรับตำแหน่งป้ายจุดยอดตามทิศกราฟ |
| สมการเขียนผิดธรรมเนียม เช่น "y = 1x² + -2x + -3" | เอาค่าตัวแปรมาต่อ string ตรงๆ ไม่ได้จัดรูปแบบ | เขียนฟังก์ชัน `format_quadratic()` กลาง จัดการเครื่องหมาย +/− และตัด "1" หน้า x² อัตโนมัติ |
| ทรงกรวย: หัวเรื่องยาวตกขอบภาพ | ข้อความหัวเรื่องยาวเกินความกว้างรูป | (ยังไม่ได้แก้ — ต้องย่อข้อความหรือขยาย figure) |
| ภาพเรขาคณิตทั่วไป ยังไม่มีชื่อจุดยอด (A,B,C), ขีดบอกด้านเท่ากัน | ยังไม่ได้ออกแบบส่วนนี้ | (ยังไม่ได้แก้ — เป็นขั้นต่อไป) |

## 8. Checklist ควบคุมคุณภาพที่แนะนำให้ใช้ต่อ

**กราฟฟังก์ชัน (พาราโบลา, sin/cos ฯลฯ):**
- [ ] สมการเขียนถูกธรรมเนียม (ไม่มี "1x²", ไม่มี "+ −")
- [ ] มีจุดยอด/จุดสำคัญกำกับ
- [ ] มีจุดตัดแกน x, y กำกับ
- [ ] ชื่อแกน + หน่วย ถูกต้องตามบริบทโจทย์
- [ ] ช่วงแกนครอบคลุมจุดสำคัญทั้งหมด ไม่บังคับ aspect ถ้าคนละหน่วย

**รูปเรขาคณิต (สามเหลี่ยม, สี่เหลี่ยม, วงกลม ฯลฯ):**
- [ ] มีชื่อจุดยอด (A, B, C หรือ ก ข ค)
- [ ] มีขีดบอกด้านเท่ากัน/สัญลักษณ์มุมฉากตามธรรมเนียม
- [ ] เส้นช่วย (เส้นสูง ฯลฯ) เป็นเส้นประ แยกจากเส้นโครงร่างหลัก
- [ ] พิกัด/จุดตัดคำนวณจากสมการจริง ไม่ hardcode

**ทุกภาพ:**
- [ ] ข้อความหัวเรื่อง/ป้ายไม่ตกขอบภาพ
- [ ] เปิดดูภาพจริงเทียบ checklist ก่อนใช้งาน (รันโค้ดผ่านไม่ได้แปลว่าภาพถูก)

## 9. สิ่งที่ยังไม่ได้ทำ / ขั้นต่อไปที่แนะนำ

1. ไล่แก้ภาพที่เหลือตาม checklist (ทรงกรวยตกขอบ, เพิ่มชื่อจุดยอดในรูปเรขาคณิตทุกรูป)
2. ลงมือสร้างระบบบันทึกผลนักเรียน + วิเคราะห์จุดอ่อนจริง (ตอนนี้มีแค่แบบร่าง pipeline)
3. ขยายคลังข้อสอบให้ครอบคลุมหัวข้ออื่นเพิ่ม
4. ถ้าต้องการโจทย์เรขาคณิตแบบสร้างรูป (สามเหลี่ยม วงกลมล้อมรอบ) พิจารณาใช้ GGBScript เสริมเฉพาะจุด (ต้องรันบนเครื่องตัวเอง ไม่ใช่ในแซนด์บ็อกซ์นี้)
